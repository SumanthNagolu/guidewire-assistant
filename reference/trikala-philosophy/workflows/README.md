# Trikala Workflows

This directory contains workflow templates showing how personas collaborate in each SDLC phase.

## Available Workflows

### Phase 1: Discovery
- **File:** `01-discovery-workflow.md`
- **Personas:** Saraswati (lead), CEO, Sadhguru
- **Goal:** Extract complete vision through 2000 questions

### Phase 4: Design
- **File:** `04-design-workflow.md`
- **Personas:** Krishna + Brahma (leads), Sadhguru
- **Goal:** Create architecture and UI designs

## Workflow Template Structure

Each workflow includes:
1. **Objective:** What we're trying to achieve
2. **Active Personas:** Who participates
3. **Duration:** Expected timeline
4. **Workflow Steps:** Detailed step-by-step process
5. **Success Criteria:** How we know we're done
6. **Artifacts Location:** Where outputs are stored

## How to Use Workflows

1. Identify current phase
2. Open relevant workflow file
3. Follow steps sequentially
4. Activate appropriate personas
5. Produce required deliverables
6. Check success criteria
7. Get approvals
8. Hand off to next phase

## Creating New Workflows

When adding new workflow:
1. Use numbering: `XX-phase-name-workflow.md`
2. Follow template structure
3. Define persona responsibilities clearly
4. Include handoff protocol
5. Document in this README

## Workflow Principles

- **Sequential execution:** Complete one step before next
- **Clear handoffs:** Explicit context passing between personas
- **Quality gates:** Sadhguru reviews at critical points
- **CEO approval:** Final sign-off on major deliverables
- **Complete documentation:** Everything logged in PROJECT-HISTORY.md
